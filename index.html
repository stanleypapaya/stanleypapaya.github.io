对 DOM 的一些理解

  1. DOM === 想象的树型结构模型.

  2. DOM提供的API的作用就是修改或者查看 HTML 代码

  3. DOM中的O指的是Object,他是在内存中,按照树型结构,通过构造函数(如Node,Element(翻译为标签比较好),Document三个构造函数),构造出对象,将 DOM 展现到内存中

  4. DOM的D指的是Document,可以认为是 HTML 文档

  5. DOM的M指的是Model,因为在 HTML结构 在内存中不好用笔表示,所以用一个模型来表示,这个模型就是树型结构

  6. DOM 树型结构

    1. 上图就是DOM,其中的每个节点(包括矩形节点和椭圆形节点)都是Node类型

    2. document节点是Document构造函数的一个实例对象,document节点代表了整个文档(整个树型结构

      ,我们可以通过直接输入document来获取document节点

    3. html节点是Element构造函数的一个实例对象,html节点又叫根节点

      我们可以通过输入document.documentElement来获得html节点

    4. 椭圆形的文本节点:”你好,我叫饶家俊” 是Text构造函数的一个实例对象(文本节点是Text构造函数的一个实例对象)

    5. Node,Element,Text的关系

  1. JavaScript将 HTML 文档渲染成 DOM 的树型结构.

Node属性

  1. 节点本身某些特征的属性:
    • Node.nodeName:
      • 看着DOM,如果你不确定某个Node节点是什么类型(不确定某个Node节点是矩形还是椭圆形)

    • Node.nodeType:
      • 根据Node类型返回某些数字
      • Element类型,返回的数字是1
      • Text类型,返回的数字是3
      • Document类型,返回的数字的9
    • Node.textContent
      • 返回的当前节点及其所有后代的文本内容
      • 值得注意的是,因为Node.textContent是Node属性,所以文本节点也是有textContent的
      • innerHTML和innerText是Element的属性,所以TextNode.innerHTML返回的是undefined,注意,并不是返回null
  2. 节点结构关系属性
    • 兄弟关系

    • 儿子关系

    • 父关系
      • Node.parentNode
    • 上面的所有关系属性都可以获取到Text节点(除了父关系)
    • Node.childNodes是最特殊的,它返回的是一个伪数组,里面是Node节点,并且伪数组内的值是动态变化的

Posted on 2018-11-06

JS数据类型分类

JavaScript中的数据类型分为数字（Number），字符串（String），布尔型（Bool），Null，Undefined和对象（Object）

Number

JavaScript 内部，所有数字都是以64位浮点数形式储存,整数也是如此。

整数范围

下例展示了Number对象所能表示的最大和最小整数 (详情请参阅 ECMAScript standard, chapter 8.5 The Number Type):

数值的进制

  • 十进制：没有前导0的数值。
  • 八进制：有前缀0o或0O的数值，或者有前导0、且只用到0-7的八个阿拉伯数字的数值。
  • 十六进制：有前缀0x或0X的数值。
  • 二进制：有前缀0b或0B的数值。

默认情况下，JavaScript 内部会自动将八进制、十六进制、二进制转为十进制

NaN

NaN是 JavaScript 的特殊值，它属于Number类型，但表示“非数字”（Not a Number）

  • 主要出现在将字符串解析成数字出错的场合
  • 另外，一些数学函数的运算结果会出现NaN

String

字符串就是零个或多个排在一起的字符，放在单引号或双引号之中。

转义符

（\）在字符串内有特殊含义，用来表示一些特殊字符，所以又称为转义符,在字符串中不占length

  • \0 ：null（\u0000）
  • \b ：后退键（\u0008）
  • \f ：换页符（\u000C）
  • \n ：换行符（\u000A）
  • \r ：回车键（\u000D）
  • \t ：制表符（\u0009）
  • \v ：垂直制表符（\u000B）
  • \’ ：单引号（\u0027）
  • \” ：双引号（\u0022）

字符串多行书写

在表达字符串时常常因为长度或者易读性问题需要换行，一般连接运算符（+）可以连接多个单行字符串，将长字符串拆成多行书写。输出时为单行

Bool

布尔值代表“真”和“假”两个状态。“真”用关键字true表示，“假”用关键字false表示。布尔值只有这两个值。

6个falsy值

在javaScript中只有以下6个值为false，其他都为true

  • 0
  • ‘’和””
  • false
  • null
  • undefined
  • NaN

Undefined 和 Null

null与undefined都可以表示“没有”，含义非常相似，但在用法上有细微的区别

  • 如果一个变量(非object）没有赋值 -undefined

  • 如果有一个对象（object）没有赋值 -null

Object

对象就是一组“键值对”（key-value）的集合，是一种无序的复合数据集合。

Heap堆排序

Posted on 2018-11-01

堆排序的过程

  • 首先让我们创建一个数组:

  • 把数组转换为完全二叉树的结构来表示

  • 比较右下角两个叶节点的数字

  • 选出较大的数字和同支上层父节点数字比较，保证大的在上层，不然则交换

  • 比较最下层相邻的两个叶节点的数字

  • 重复相同的步骤

  • 直到底下两层都满足最大堆

  • 在倒数第二层，比较最右边两个节点的数字，重复相同的过程

  • 被换到倒数第二层的数字，仍要和同支最下层的数字比较，保证大的数字在上层


  • 如此不断重复，直到最大的数字被换到最上层的父节点, 并且下几层都满足最大堆

  • 将最上层父节点的数字和右下角的数字交替位置，并拿出二叉树结构

  • 在数组中最大的数字与最后一个数字交换位置

  • 将二叉树再进行一次最大堆调整，保证最大的数字在最上层

  • 再次将最上层父节点的数字和右下角的数字交替位置，并在数组中交换位置，将最大的数字拿出二叉树结构

  • 如此不断重复上述过程，直到所有数字都被拿出二叉树，数字也排序完成

WWW的历史

1990年

万维网（world wide web）诞生
Tim Berners-Lee 发明了用网址就能访问页面的方法，他发明了第一个网页，第一个浏览器和服务器

1990年之前

  • 1971年， Email开始使用@符号
  • 1979年， 邮件讨论组被发明出来，人们在一个话题下可以公开互发邮件
  • 1980年， HTTP被提出

HTTP请求的组成

  1. 动词 路径 协议/版本
  2. Key1: value1
  3. Key2: value2
  4. Key3: value3
  5. Content-Type: application/x-www-form-urlencoded
  6. Host: www.baidu.com
  7. User-Agent: curl/7.54.0
  9. 要上传的数据

请求最多包含4个部分（POST），最少包含3个部分（如GET），第三部分永远是一个回车（\n)

HTTP响应的组成

  2. Key1: value1
  3. Key2: value2
  4. Content-Length: 17931
  5. Content-Type: text/html

常见git命令的使用

  • git init

    • 将当前目录即demo-1初始化为一个本地仓库
    • 该仓库主要用于远端服务器管理代码
    • 保存git历史提交的版本信息，防止冲突

  • git add .

    • 将当前目录所有变动的文件加到暂存区
    • 查看状态后发现文件名左边有绿色A
    • 绿色A表示文件已经被加到暂存区

  • git commit -v

    • 将暂存区的文件提交到.git仓库
    • 表示三个文件（index.html, main.js, style.css)被提交
    • 提交的信息是‘first commit’
